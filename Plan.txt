Phase 2.5 — Drift Simulation Engine (1-2 days)

Sits between Phase 4 and Phase 5 in execution order but documented separately since it's a standalone concept.

What you build:

src/drift_simulator.py

simulate_drift(df, drift_intensity) function
Shifts amount distribution upward by drift_intensity * 60%
Shifts fraud rate from 3.5% toward 7-8%
Increases velocity_1hr by drift_intensity * 2x
Shifts hour distribution of fraud transactions toward daytime
Introduces new merchant_tx_count patterns
Fully parameterized — intensity from config.yaml


scripts/generate_inference_batch.py

Reads drift_intensity from config/config.yaml
Takes test set, applies simulate_drift()
Saves output to data/inference_batches/batch_YYYYMMDD.csv
After successful retraining — increments drift_intensity in config, then resets to base after retrain


tests/test_drift_simulator.py

Test that shifted distributions are statistically different from original
Test intensity scaling
Test config read/write



What you learn: Statistical distribution shifting, parameterized simulation design

Phase 5 — Drift Detection + Auto Retrain (3-4 days)
What you build:

src/drift_detector.py

Load latest inference batch from data/inference_batches/
Load training data baseline
Evidently DataDriftPreset report
KS test on numeric features — amount, velocity_1hr, user_avg_amount_30d
Chi-square on categoricals — card_type, hour_bucket
Flag drift if any feature p-value < 0.05 (threshold from config)
Save HTML report to reports/
Log all scores + p-values to PostgreSQL drift_logs table


src/retraining_pipeline.py

Calls ingestion → feature engineering → training → evaluation in sequence
Loads current Production model F1 from PostgreSQL
Compares new model F1 vs current
If new F1 > old F1 by >1% — promote new model to Production in MLflow registry
Else — keep current model, log warning
Safety checks:

File lock — prevent simultaneous retraining
Minimum 6hr gap between retrains
Max 3 retrains per day — log alert if exceeded


Log full event to logs/retraining_history.json + PostgreSQL retrain_logs


tests/test_drift_detector.py

GitHub Actions: retrain.yml — daily cron, calls generate_inference_batch.py then drift_detector.py, triggers retraining_pipeline.py if drift detected
What you learn: Statistical drift tests, Evidently reports, safe retraining orchestration
